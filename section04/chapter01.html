React.js란?

Meta(Facebook)이 개발한 오픈소스 JavaScript 라이브러리
대규모 웹 서비스의 UI를 더 편하게 개발하기 위해 만들어진 기술로 현재 가장 많은 인기를 누리고 있음.
React.js로 만들어진 서비스들 : 넷플릭스, 페이스북, 인스타그램, 노션

React의 기술적인 특징

1. 컴포넌트를 기반으로 UI를 표현한다
2. 화면 업데이트 구현이 쉽다
3. 화면 업데이트가 빠르게 처리된다
모든 내용을 완벽하게 이해할 필요가 없이 먼저 훑어보고 완강 후 다시 복습하는 것을 권장.

1. 컴포넌트를 기반으로 UI를 표현한다

컴포넌트(Component)란? : 우리말로 "구성요소" 라는 뜻이며 화면을 구성하는 요소, UI를 구성하는 요소를 말함
만약 웹페이지의 구성이 Header, Main, Footer로 구성된다면, 해당 화면을 보여주는 요소를 각각의 파일로 나누어 개발 후 사용하기 때문에
어떻게 보면 레고를 조립하는 느낌으로 개발할 수 있다.
컴포넌트화를 하지 않고 동일한 구성의 페이지들을 만들면, 중복된 코드가 많아지기 때문에 페이지의 수가 늘어날 수록 코드를 수정하기 어려워 진다.

2. 화면 업데이트 구현이 쉽다

업데이트란? : 사용자의 행동(클릭, 드래그)에 따라 웹 페이지가 스스로 모습을 바꿔 상호작용 하는 것
React.js에서는 화면 업데이트를 구현하기 쉽다 => 그 이유는 리엑트가 선언형 프로그래밍 방식으로 동작하기 때문이다.
** 선언형 프로그래밍이란? : 과정은 생략하고 목적만 간결히 명시하는 방법 (마치 우리가 식당에서 메뉴 주문하는 것과 일치)
** 명령형 프로그래밍이란? : 목적을 이루기 위한 모든 일련의 과정을 설명하는 방식 (음식을 주문한다면 조리법을 전부 다 지정해주는 식으로 주문) [JavaScript만 쓴다면 여기에 해당]

선언형 프로그래밍을 다시 정리하자면, 업데이트를 위한 복잡한 동작을 직접 정의할 필요 없이 특정 변수의 값을 바꾸는 것 만으로도 화면을 업데이트 시킬 수 있다.

3. 화면 업데이트가 빠르게 처리된다

화면 업데이트를 쉽게 구현할 수 있으면서 동시에 빠르게도 처리해 준다.
브라우저의 렌더링 과정에 대해 알아야 한다.
DOM(Document Object Model) 이란? : HTML의 문서를 일종의 객체 모델로 변환한 것입니다. 브라우저가 더 쉽게 이해하도록 변환한 것
CSSOM(CSS Object Model) 이란? : CSS 코드를 브라우저가 더 쉽게 이해하도록 변환한 것

위의 과정들이 끝나면 DOM과 CSSOM을 합쳐서 Render Tree를 만드는데, 이는 웹 페이지의 설계도 역할을 한다.
** DOM에는 요소들의 위치, 배치, 모양에 관한 모든 정보가 담김
** CSSOM에는 요소들의 스타일과 관련된 모든 정보가 담김
그렇기 때문에 Render Tree에는 화면에 나와야 하는 모든 요소들에 대한 정보들이 담긴다.

마지막으로 Painting이라는 작업을 진행하며, 이는 모든 요소들을 실제로 화면에 그려내는 과정이라고 할 수 있다.

그렇다면 화면의 업데이트는 어떻게 일어날까? : JavaScript가 DOM을 수정하면 업데이트가 발생한다.
